# データモデル

## 概要

この文書は，Geshi のデータモデル仕様の正本である．

Geshi が扱う `podcast`，`streaming`，`feed` の共通要素と差分を，ここで管理する．

## 運用

- データモデル仕様の通常の更新は，この文書に対して行う
- データモデルの大きな前提や位置づけを変更する場合は，新しい ADR で扱う

## 対象

この文書では，少なくとも次のようなモデルを扱う想定である．

- `source`
- `channel`
- `feed`
- `program`
- `episode`
- `article`
- `asset`
- `transcript`
- `summary`

## 各モデルの概要

### `source`

収集対象の大きな出所を表す．

たとえば podcast 配信元，streaming の放送元，feed のサイトやサービスのような，「どこから取ってくるか」の単位である．

### `channel`

継続的に追跡する公開単位を表す．

podcast 番組，streaming のチャンネルや番組枠，RSS feed など，更新を継続監視する対象を表現するためのモデルである．

### `feed`

更新情報を取得するための配信単位を表す．

RSS / Atom などの配信 URL や，それに準ずる更新取得の入口を保持する．`channel` と 1 対 1 とは限らず，更新取得のための技術的な単位として扱う．

### `program`

streaming や放送型コンテンツにおける編成上の番組を表す．

定期番組や放送枠のように，個々の配信回より一段上の単位が必要な場合に使う．podcast や feed では不要な場合もある．

### `episode`

個別の配信回，公開回，録画対象回のような，時系列上の 1 件を表す．

podcast の各回，streaming の各放送回，あるいは公開日時を持つ個別コンテンツを共通的に表現する中心モデル候補である．

### `article`

feed における個別記事を表す．

記事本文，記事 URL，公開日時，更新日時など，テキスト記事としての実体を保持するためのモデルである．

### `asset`

保存対象となる実ファイルや外部実体を表す．

音声ファイル，録画ファイル，取得した HTML，サムネイルなど，`episode` や `article` にひもづく実体データを表現する．

### `transcript`

音声・動画コンテンツから得られた文字起こし結果を表す．

Geshi 自体は文字起こしエンジンを持たず，外部 API の結果をこのモデルとして保存し，再生や検索に利用する．

### `summary`

`episode` や `article` などに対して生成された要約や補助テキストを表す．

必須モデルとは限らないが，文字起こしや本文から派生した二次的な説明情報を保持するための候補である．
