# E2E Foundation

この開発項目では，Web UI を起点として `podcast rss source` を登録し，backend / worker / storage を含む実行環境上で，しばらく待つと収集結果の `content` 一覧を確認できる最小の E2E 検証経路を，自動テストとして利用可能な状態まで実装することを受け入れ条件とする．

## 受け入れ条件

- E2E テスト導入に必要な前提判断が文書化されている
- E2E テストを実行するための最小構成と起動方法が，リポジトリ内で再現可能な形に整理されている
- 正常系では，Web UI から `podcast rss source` を登録できる
- 正常系では，登録した source に対して observe job が実行され，`content` 一覧に収集結果が反映される
- E2E テストは，frontend，api backend，worker の境界をまたぐ最小経路を自動で検証できる
- E2E テストは，外部ネットワークや不安定な外部 source に依存せず，ローカルに再現可能である
- テスト失敗時に，少なくともどの段階（起動，登録，クロール，表示確認）で失敗したかを追跡できる
- 継続クロール，acquire 後の asset 保存確認，詳細な異常系の網羅は，この開発項目の受け入れ条件には含めない

## 確認方法

- E2E テスト導入に必要な文書が `docs/acceptance/`，`docs/decisions/`，必要に応じて関連文書に揃っていることを確認する
- E2E 用の起動手順に従ってテスト実行環境を立ち上げられることを確認する
- 自動テストで，Web UI から `podcast rss source` を登録し，その後 `content` 一覧に収集結果が表示されることを確認する
- テストが frontend，api backend，worker をまたいで実行されることを確認する
- 外部依存を切り離した状態でも，同じ E2E テストが再現可能であることを確認する
