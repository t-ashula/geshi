# Geshi Concept

## 概要

Geshi は，個人用のメディアアーカイブである．

podcast，streaming，feed を継続的に収集し，あとから再生・閲覧・検索できる状態で手元に蓄積することを目的とする．

これは，新しいプロダクトを一から考えるというより，既存の `net-radio-archive` 的な用途を，現在の前提に合わせて作り直すためのモダナイズでもある．

## 対象

Geshi が主に扱う対象は次の 3 つである．

- podcast
- streaming
- feed

podcast と streaming については，収集だけでなく，再生，文字起こし，全文検索までを扱う．

feed については，RSS feed の取得だけでなく，可能な限り本文まで取得し，保存し，全文検索できることを前提にする．

## コア価値

Geshi のコア価値は次の通りである．

- 継続的にコンテンツを収集できること
- 手元に保存し，後から再利用できること
- 音声コンテンツを文字起こしして検索可能にできること
- feed の記事を本文込みで保存し，検索できること
- 過去のコンテンツを，アーカイブとして横断的に参照できること

単なるダウンローダや録画ツールではなく，「あとから探して使える個人用アーカイブ」であることが重要である．

## 想定ユーザー

基本的に一人用である．

自分が追っている podcast，streaming，feed を自分のために集め，自分のために保存し，自分のために検索できることを優先する．

## 操作面

Geshi の操作面は 2 つを想定する．

- CLI
  - 管理系の入口
- Web UI
  - 閲覧系の入口

Web UI では少なくとも次のような性格を持つ画面を想定する．

- 番組表的な UI
- メーラー的な UI
- 管理画面

## 境界

Geshi は，収集，保存，再生/閲覧，検索，文字起こし結果の利用を担う．

一方で，文字起こしエンジンそのものはこのプロジェクトでは実装しない．文字起こしは外部 API を利用し，Geshi はその結果を保存して使う側に留まる．

## 非目標

現時点での非目標は次の通りである．

- 多人数向けサービスにすること
- SaaS 化を前提にすること
- 汎用クローラ基盤を作ること
- 文字起こしモデル自体をこのリポジトリで実装・運用すること

## 一文で言うと

Geshi は，podcast，streaming，feed を対象に，収集から保存，検索，再利用までを一貫して支える，一人用の現代的な `net-radio-archive` である．
