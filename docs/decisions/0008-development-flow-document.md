# ADR-0008: 開発フローの正本を専用文書へ切り出す

## ステータス

決定

## 範囲

全体

## コンテキスト

- ADR-0006 によって，Geshi の開発フロー自体はすでに決定されている
- 一方で，開発フローは設計判断というより運用文書に近く，実態に合わせて比較的柔軟に更新したい
- `決定` になった ADR の本文は原則として大きく書き換えない運用であり，フローの細かな改善を ADR の本文に積み増していくのは扱いづらい
- そのため，開発フローの内容そのものと，「どこを正本とするか」という設計判断を分けたい

## 決定

- 開発フローの正本は [`docs/development-flow.md`](../development-flow.md) とする
- ADR-0006 で定めた開発フローの内容は，`docs/development-flow.md` に切り出して管理する
- 今後，開発フローの通常の更新は `docs/development-flow.md` に対して行う
- 開発フローの運用上の大きな前提や位置づけを変更する場合は，新しい ADR で扱う

## 影響

- 開発フローの細かな改善を，`決定` 済み ADR を直接編集せずに行いやすくなる
- ADR は「開発フローを専用文書で管理する」という判断を保持し，運用文書の本文は別ファイルで維持できる
- `docs/development-flow.md` を参照すれば，現行の開発フローを 1 か所で確認できる

## 代替案

- ADR-0006 をそのまま正本として使い続ける
  - 決定済み ADR の更新に慎重さが必要なため，運用文書としては硬すぎる
- ADR-0006 を直接書き換えて追従する
  - ADR の履歴性と運用文書の可変性が混ざる

## 備考

- ADR-0006 自体は，開発フローを採用した事実の記録として残す
- `docs/development-flow.md` の初期内容は，ADR-0006 の決定内容をそのまま移したものとする
