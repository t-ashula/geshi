# ADR-0016: frontend の UI フレームワークを選定する

## ステータス

決定

## 範囲

`frontend/`

## コンテキスト

- ADR-0003 により，`frontend/` は閲覧，再生，検索，管理の UI 層として扱うことが決まっている
- backend 側では HTTP API フレームワークとして Hono を採用し，API 表現の基本方針も固まりつつある
- 次に `frontend/` を具体化するには，UI フレームワークを選定してよい段階に入っている
- 番組表的 UI，メーラー的 UI，管理画面，検索画面を考えると，状態遷移や画面構成を素直に扱えるものを選びたい

## 決定

- frontend の UI フレームワークとして Vue を採用する
- 初期段階では，小さく始めることを優先する
- ルーティング，状態管理，データ取得補助，UI コンポーネントライブラリは，最初から一括導入せず，必要になった時点で追加する
- backend API との接続は，まずは薄い API client を前提に考える
- 採用理由は次の通りである
  - TypeScript / Vite と組み合わせやすい
  - 管理画面や一覧中心の UI を構成しやすい
  - backend API と素直につなぎやすい
  - 個人開発で継続しやすい感覚に合う

## 影響

- frontend 実装の具体化を進めやすくなる
- 後続の画面モデル設計やデータ取得方式の整理に進みやすくなる
- Vue Router や状態管理ライブラリを，必要になった時点で判断しやすくなる

## 代替案

- React を採用する
  - 定番だが，今回の frontend では Vue の方が進めやすいと判断した
- UI フレームワーク選定をさらに後ろへ送る
  - 画面設計の具体化が進みにくい
- ルーティングや状態管理まで一度に選ぶ
  - 判断対象が広がりすぎる

## 備考

- 本 ADR は，frontend の UI フレームワークとして Vue を採用する判断を記録する
- `npm install` や初期実装は後続の開発項目で扱う
