# ADR-0013: search の責務と方式を定める

## ステータス

決定

## 範囲

全体

## コンテキスト

- Geshi の concept では，podcast，streaming，feed を横断して後から検索できることをコア価値としている
- `docs/models.md` では，`episode`，`asset`，`derivative` を中心に概念モデルを整理した
- ADR-0011 により，永続化では元データと再構築可能な派生物を分けて考える方針になっている
- この段階で，検索機能が何を責務として持ち，そのための検索用データをどう位置づけるかの基本方針を定めておきたい

## 決定

- Geshi の検索機能は，横断的な全文検索と参照補助を担う
- 検索のためのインデックスや付随データは，保存の中心ではなく，元データから再構築可能な派生物として扱う
- 検索機能の対象には，少なくとも次を含められるようにする
  - `episode` の基本メタデータ
  - `asset` のうち本文テキストや説明文として扱うもの
  - `derivative` のうち `transcript`，`summary`，`notes` などのテキスト系データ
- 検索機能は，媒体種別ごとの内部差異をできるだけ吸収し，利用側からは横断検索として扱えるようにする
- 検索インデックスの具体方式，ランキング，検索 UI の詳細は本 ADR では決めない

## 影響

- 検索機能と検索用データの役割を分けて考えやすくなる
- 検索用データを保存の中心と混同せず，永続化方針と整合した設計にしやすくなる
- `episode / asset / derivative` のどこまでを検索対象にするかの基本線を共有できる
- 後続の検索基盤選定や UI 設計を進めやすくなる

## 代替案

- 検索機能を単なる `episode` メタデータ検索に限定する
  - transcript や本文検索の価値を十分に出しにくい
- 検索用データを保存の中心に近いものとして扱う
  - 永続化責務が重くなりやすい

## 備考

- 本 ADR は，検索機能の責務と検索対象の基本方針を定める
- 検索基盤やインデックス更新方式の具体選定は後続 ADR で扱う
