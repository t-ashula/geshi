# ADR-0002: DB 設計ポリシー

## ステータス

決定

## 範囲

geshi/model

## コンテキスト

- スキーマ設計で命名の不一致が生じやすく、明示的な規約が必要だった
- ORM（Prisma）上の命名と実 DB 上の命名を整理しないと、実装時の混乱とメンテナンスコストが増える

## 決定

- テーブル名はすべて小文字の複数形とする
- カラム名はすべて `snake_case` とする
- Prisma 上のモデル名・プロパティ名は Prisma の標準スタイルを採用する
- Prisma のマッピング機能で、Prisma 側の命名と DB 側の命名を対応付ける

## 影響

- 命名規則が一貫し、スキーマとアプリケーションコードの対応が明確になる
- データアクセス実装・スキーマ生成の自動化との互換性が高くなる
- 既存実装で命名が異なる箇所は、段階的な移行またはマッピング調整が必要になる

## 代替案

- キャメルケースや PascalCase を DB 命名に採用する
- テーブル名を単数形にする
